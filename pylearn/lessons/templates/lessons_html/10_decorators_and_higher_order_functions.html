Decorators and Higher-Order Functions in Python

Python provides powerful features like decorators and higher-order functions that allow for more concise and reusable code.

Understanding Decorators

Decorators are a way to modify or extend the behavior of functions or methods without changing their source code. They are commonly used for logging, access control, and memoization.

@decorator_name
def my_function():
    # function code

Commonly Used Decorators

  • @staticmethod: Converts a method into a static method.
  • @classmethod: Allows a method to access class variables and methods.
  • @property: Converts a method into a property.

Higher-Order Functions

Higher-order functions are functions that either accept other functions as arguments or return them as results.

def apply_operation(a, b, operation):
    return operation(a, b)

result = apply_operation(5, 3, lambda x, y: x + y)
print(result)  # Outputs 8

Combining Decorators with Higher-Order Functions

Decorators can be created using higher-order functions to extend their capabilities, making your code cleaner and more reusable.

def decorator_with_args(arg):
    def actual_decorator(func):
        def wrapper(*args, **kwargs):
            print(f"Argument: {arg}")
            return func(*args, **kwargs)
        return wrapper
    return actual_decorator

@decorator_with_args("some argument")
def sample_function():
    print("Sample function executed")

sample_function()
